Principal Component Analysis (PCA) with cuML

In[5]:
# Import necessary libraries
import numpy as np
import cuml
from cuml.decomposition import PCA as cuPCA
from sklearn.decomposition import PCA as skPCA
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import time

Create and Load the synthetic data

In[6]:
# Create a synthetic dataset with 300,000 samples and 20 features
# This will serve as a large dataset for testing the performance of cuML and scikit-learn PCA.
n_samples = 300000
n_features = 20
X = np.random.rand(n_samples, n_features)

Normalize the dataset (mean = 0, variance = 1)

In[7]:
# Normalize the dataset (mean = 0, variance = 1) using StandardScaler
# PCA is sensitive to the scale of the data, so we standardize the dataset.
scaler = StandardScaler()
X_normalized = scaler.fit_transform(X)

Apply PCA using cuML to reduce the dimensionality to 3 principal components.

In[12]:
# Apply PCA using cuML
# Initialize the cuML PCA object to reduce the dataset to 3 principal components.
cuml_pca = cuPCA(n_components=3)
In[13]:
# Measure the time taken by cuML PCA
start_time = time.time()
X_cuml_pca = cuml_pca.fit_transform(X_normalized)
cuml_time = time.time() - start_time
print(f"cuML PCA time: {cuml_time:.4f} seconds")

cuML PCA time: 0.6322 seconds

Apply PCA using scikit learn

In[14]:
# Apply PCA using scikit-learn for comparison
# Initialize the scikit-learn PCA object to reduce the dataset to 3 principal components.

Overall observations and Summary

Summary of cuML vs. scikit-learn PCA Execution Time Comparison:

1. Execution Time:

  • cuML PCA: Significantly faster due to GPU acceleration i.e 0.6322 seconds. Likely took a fraction of the time compared to scikit-learn.
  • scikit-learn PCA: Took 1.4027 seconds, typical for large datasets on a CPU.

2. Performance:

  • cuML is much faster, especially for large datasets, by utilizing parallel processing on the GPU.
  • scikit-learn is slower but more accessible for environments without a GPU.

3. Accuracy:

  • Both cuML and scikit-learn PCA produce equally accurate results with no loss in precision.

4. Use Cases:

  • cuML PCA: Best for large datasets and when speed is crucial (requires an NVIDIA GPU).
  • scikit-learn PCA: Suitable for smaller datasets or CPU-only environments.

Finally, with our results and generally cuML is ideal for large datasets with GPUs, offering major speed improvements, while scikit-learn remains practical for smaller or CPU-based tasks.
